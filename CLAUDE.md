# Morning Motivation App

## Project Overview
A Progressive Web App (PWA) that replaces morning phone scrolling with curated, meaningful content. The app displays daily content based on the current date.

## Daily Content Features
1. **Historical Battle** - A battle that occurred on this day in history
2. **Catholic Daily Gospel** - The gospel reading for the day
3. **Saint's Feast Day** - Catholic saint celebrated on this day
4. **Famous American Quote** - Rotating quotes from: Theodore Roosevelt, Hank Williams Jr, Mike Tyson, and similar figures
5. **Artist Portfolio** - Featured artwork from masters like Dalí, Picasso, Van Gogh

## Tech Stack
- **Framework**: Next.js 14+ (App Router)
- **Styling**: Tailwind CSS
- **PWA**: next-pwa package
- **Deployment**: Vercel (free tier)
- **Data**: Mix of free APIs and static JSON files

## Data Sources (All Free)

### Historical Events/Battles
- Wikipedia "On This Day" API: `https://en.wikipedia.org/api/rest_v1/feed/onthisday/events/{MM}/{DD}`
- Filter results for battles/military events using keywords

### Catholic Gospel
- Option 1: Evangelizo API - `https://feed.evangelizo.org/v2/reader.php`
- Option 2: Static Lectionary JSON mapping dates to readings + Bible API (`https://bible-api.com/`)
- Use the Lectionary cycle (A, B, C for Sundays; I, II for weekdays)

### Saints Feast Days
- Static JSON file curated from public liturgical calendar
- Source data: Catholic Church's General Roman Calendar

### Famous American Quotes
- Static JSON file (`/data/quotes.json`)
- Curated collection - rotate randomly or by day of year
- Include: Theodore Roosevelt, Hank Williams Jr, Mike Tyson, Benjamin Franklin, General Patton, etc.

### Artist Portfolios
- Art Institute of Chicago API (free, no key): `https://api.artic.edu/api/v1/artworks`
- Filter by artist name, get high-quality images
- Alternative: Metropolitan Museum of Art API

## PWA Requirements
The app must include:
1. `manifest.json` with app name, icons, theme colors, display: "standalone"
2. Service worker for offline capability (next-pwa handles this)
3. Meta tags in layout for iOS compatibility
4. Icons in multiple sizes (at minimum: 192x192, 512x512)

## Design Guidelines
- **Theme**: Dark mode by default (easy on eyes for morning)
- **Typography**: Clean, readable, generous spacing
- **Layout**: Mobile-first, single column, scrollable cards
- **Colors**: Warm accent color (orange/gold) against dark background
- **Feel**: Calm, focused, no distractions

## User Experience
1. User opens app (PWA from home screen)
2. App displays current date prominently
3. Content sections load with today's data
4. Each section is a card that can be expanded for more detail
5. Minimal interaction required - just read and absorb

## Development Commands
```bash
npm run dev      # Local development
npm run build    # Production build
npm run start    # Start production server
```

## Deployment
1. Push to GitHub repository
2. Connect repo to Vercel
3. Vercel auto-deploys on every push
4. Access via Vercel URL or custom domain

## PWA Installation
- **iOS**: Safari → Share → "Add to Home Screen"
- **Android**: Chrome → Menu → "Install App"

## Notes
- All external API calls should have error handling and fallbacks
- Cache API responses where possible to reduce load times
- Static JSON data should be comprehensive enough to always have content
- Date handling must account for timezones (use user's local date)
